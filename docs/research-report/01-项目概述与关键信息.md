## 1.1 项目简介

**NostalgiaForInfinity (NFI)** 是一个为 Freqtrade 量化交易框架设计的高级加密货币交易策略。它实现了多模式交易系统，支持多头（Long）和空头（Short）双向交易，适应各种市场条件。

### 核心特征

| 特征 | 值 | 说明 |
|------|-----|------|
| **主策略文件** | `NostalgiaForInfinityX7.py` | 约73,000行，最新版本 |
| **必需时间框架** | 5分钟 (`timeframe = "5m"`) | **不可修改** |
| **多时间框架分析** | 5m, 15m, 1h, 4h, 1d | 加 BTC 参考数据 |
| **启动蜡烛数** | 800 | 指标预热所需 |
| **交易模式** | 14种 | 7种多头 + 7种空头 |
| **头寸管理** | Grinding + Rebuy + Derisking | 三层风险管理 |

### 版本历史

项目存在多个策略版本（X 到 X7），**X7 是当前活跃开发版本**：
- 信号调优提交遵循模式："signal X: fine tune"
- 所有更改通过自动化回测验证

## 1.2 核心架构

### 生命周期方法

```
┌─────────────────────────────────────────────────────────────┐
│                    Freqtrade 交易引擎                        │
├─────────────────────────────────────────────────────────────┤
│  populate_indicators()  ← 计算5m + 合并15m/1h/4h/1d + BTC指标 │
│         ↓                                                    │
│  populate_entry_trend() ← 评估条件 → 分配 enter_tag          │
│         ↓                                                    │
│  交易执行 → adjust_trade_position() (DCA/获利了结)           │
│         ↓                                                    │
│  custom_exit() → 根据 enter_tag 路由 → 模式特定出场方法       │
└─────────────────────────────────────────────────────────────┘
```

### 关键方法说明

| 方法 | 用途 |
|------|------|
| `populate_indicators()` | 计算所有时间框架的技术指标 |
| `populate_entry_trend()` | 生成带模式标签的多头/空头入场信号 |
| `populate_exit_trend()` | 生成出场信号 |
| `custom_exit()` | 通过 `enter_tag` 按交易模式路由出场 |
| `adjust_trade_position()` | 处理DCA（摊低成本）和获利了结 |
| `custom_stake_amount()` | 动态计算头寸大小 |

### 数据流详解

```
数据摄取 (交易所 OHLCV)
    ↓
指标计算
    ├── 基础5m指标 (RSI, EMA, BB, MFI, CMF等)
    ├── 多时间框架指标 (15m, 1h, 4h, 1d)
    └── BTC参考指标 (跨资产相关性)
    ↓
信号生成 (基于配置的条件和指标组合)
    ↓
头寸管理 (Grinding/Rebuy/Derisking)
    ↓
订单执行 (通过Freqtrade引擎)
```

## 1.4 必需的系统要求

### 硬件要求

| 级别 | RAM | CPU | 存储 | 说明 |
|------|-----|-----|------|------|
| **最小** | 2GB | 1核 | 50GB SSD | 基本运行 |
| **推荐** | 4GB+ | 2核+ | 100GB SSD | 稳定运行 |
| **优化** | 8GB+ | 4核+ | 200GB SSD | 高性能 |

### 软件要求

- **Python**: 3.10+
- **Freqtrade**: 最新稳定版
- **Docker**: 可选但推荐
- **操作系统**: Linux/macOS/Windows

### 必需的 Freqtrade 配置

```json
{
  "use_exit_signal": true,
  "exit_profit_only": false,
  "ignore_roi_if_entry_signal": true
}
```

> ⚠️ **警告**: 以上三项配置**必须**按此设置，否则策略将无法正常工作。

### 网络要求

- 稳定的互联网连接
- 低延迟到交易所 API
- 建议使用 VPS 或云服务器
- 可选：交易所 co-location
