## 7.1 常见问题与解决方案

### 问题1: 时间框架覆盖

**症状**:
- 信号异常
- 交易行为不符合预期
- 回测结果与实盘差异大

**原因**:
- 后续配置文件覆盖了 `timeframe` 参数
- 策略文件中硬编码的时间框架被覆盖

**解决方案**:
```bash
# 检查最终配置
freqtrade list-config --all

# 确保 timeframe 为 5m
# 检查所有配置文件的加载顺序
```

**预防措施**:
- 在最后加载的配置中显式设置 `timeframe = "5m"`
- 不在用户配置中修改 timeframe

---

### 问题2: 配置加载失败

**症状**:
- 策略启动崩溃
- 报错: "Invalid configuration"
- 部分参数未生效

**原因**:
- JSON 语法错误
- 文件路径错误
- 参数类型不匹配

**解决方案**:
```bash
# 验证 JSON 语法
python -c "import json; json.load(open('config.json'))"

# 验证配置
freqtrade validate-config -c config.json

# 检查文件路径
ls -la configs/
```

**常见 JSON 错误**:
```json
// 错误: 尾部逗号
{
  "key1": "value1",
  "key2": "value2",  // ← 移除这个逗号
}

// 错误: 单引号
{
  'key': 'value'  // ← 改用双引号
}
```

---

### 问题3: 出场信号无效

**症状**:
- 交易卡住无法出场
- 止损不触发
- 利润目标不触发

**原因**:
- `use_exit_signal = false`
- `exit_profit_only = true`
- 出场条件从未满足

**解决方案**:
```json
{
  "use_exit_signal": true,
  "exit_profit_only": false,
  "ignore_roi_if_entry_signal": true
}
```

**验证配置**:
```bash
freqtrade list-config -c config.json | grep exit
```

---

### 问题4: 黑名单不生效

**症状**:
- 仍然交易被排除的币种
- 杠杆代币进入交易
- 稳定币被交易

**原因**:
- 黑名单文件路径错误
- 正则表达式语法错误
- 配置加载顺序问题

**解决方案**:
```bash
# 检查文件路径
ls -la configs/blacklist-*.json

# 验证 JSON
python -c "import json; print(json.load(open('configs/blacklist-binance.json')))"

# 测试正则表达式
python -c "import re; print(re.match('(BNB)/.*', 'BNB/USDT'))"
```

**正则表达式测试**:
```python
import re

patterns = [
    "(BNB)/.*",
    ".*BULL/.*",
    ".*3L/.*"
]

test_pairs = ["BNB/USDT", "ETHBULL/USDT", "BTC3L/USDT", "ETH/USDT"]

for pair in test_pairs:
    for pattern in patterns:
        if re.match(pattern, pair):
            print(f"{pair} 被 {pattern} 匹配")
            break
    else:
        print(f"{pair} 未被匹配")
```

### 问题5: Rebuy 不触发

**症状**:
- 亏损头寸未追加买入
- 平均成本未改善
- Rebuy 日志无记录

**原因**:
- `rebuy_mode_min_free_slots` 不足
- 指标确认条件未满足
- `position_adjustment_enable = False`

**解决方案**:
```python
# 检查参数
position_adjustment_enable = True
rebuy_mode_min_free_slots = 2

# 检查当前开仓数是否超过限制
# max_open_trades - current_trades >= rebuy_mode_min_free_slots
```

**调试步骤**:
1. 检查日志中的 Rebuy 相关条目
2. 验证 RSI/ROC/EMA 条件
3. 确认空闲槽位数量

---

### 问题6: Grinding 不执行

**症状**:
- 磨削未触发
- 头寸未追加
- 平均成本未降低

**原因**:
- `grinding_enable = False`
- 亏损未达到阈值
- 磨削级别耗尽

**解决方案**:
```python
# 确认开关
grinding_enable = True
position_adjustment_enable = True

# 检查阈值
grind_1_sub_thresholds_spot = [-0.12, -0.16, -0.20]
# 当前亏损必须超过 12% 才触发第一级
```

---

### 问题7: API 连接问题

**症状**:
- 连接超时
- 频繁断连
- 订单执行失败

**原因**:
- 网络不稳定
- API 速率限制
- 交易所维护

**解决方案**:
```json
{
  "exchange": {
    "ccxt_config": {
      "enableRateLimit": true,
      "rateLimit": 200,
      "timeout": 30000
    }
  }
}
```

**网络诊断**:
```bash
# 测试连接
ping api.binance.com

# 检查 DNS
nslookup api.binance.com

# 测试 API
curl -s "https://api.binance.com/api/v3/ping"
```

---

### 问题8: 内存不足

**症状**:
- 进程被杀死
- "Out of memory" 错误
- 系统变慢

**原因**:
- 交易对过多
- 数据积累过多
- 内存泄漏

**解决方案**:
```bash
# 减少交易对数量
# pairlist: number_assets: 40 (从80减少)

# 清理旧数据
rm -rf user_data/data/binance/*_1m.json
rm -rf user_data/data/binance/*_15m.json

# 监控内存
watch -n 5 free -h
```

## 7.3 日志调试指南

### 启用详细日志

```bash
# 命令行方式
freqtrade trade --strategy NostalgiaForInfinityX6 --loglevel DEBUG

# 配置文件方式
{
  "internals": {
    "process_throttle_secs": 5,
    "heartbeat_interval": 60
  }
}
```

### 日志级别说明

| 级别 | 用途 | 信息量 |
|------|------|--------|
| ERROR | 仅错误 | 最少 |
| WARNING | 警告+错误 | 少 |
| INFO | 一般信息（默认） | 中 |
| DEBUG | 详细调试 | 多 |

### 关键日志位置

```bash
# 主日志文件
user_data/logs/freqtrade.log

# Docker 日志
docker compose logs -f

# 查看最新日志
tail -f user_data/logs/freqtrade.log

# 搜索特定内容
grep "ERROR" user_data/logs/freqtrade.log
grep "rebuy" user_data/logs/freqtrade.log
grep "grind" user_data/logs/freqtrade.log
```

### 日志分析技巧

```bash
# 统计错误类型
grep "ERROR" user_data/logs/freqtrade.log | cut -d' ' -f5- | sort | uniq -c | sort -rn

# 查看特定交易对日志
grep "BTC/USDT" user_data/logs/freqtrade.log

# 查看特定时间段
grep "2024-01-15 10:" user_data/logs/freqtrade.log
```

## 7.5 紧急恢复

### 强制停止交易

```bash
# 停止 Freqtrade
pkill -f freqtrade

# Docker 方式
docker compose down

# 验证已停止
ps aux | grep freqtrade
```

### 强制平仓

```bash
# 使用 Freqtrade 命令
freqtrade forceexit --all -c config.json

# 或通过交易所界面手动平仓
```

### 数据恢复

```bash
# 备份当前状态
cp -r user_data/tradesv3.sqlite user_data/tradesv3.sqlite.bak

# 恢复备份
cp user_data/tradesv3.sqlite.bak user_data/tradesv3.sqlite
```
