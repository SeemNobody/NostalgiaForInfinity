## 6.3 安全配置建议

### API 密钥管理

#### 方法1: 单独配置文件

创建 `exampleconfig_secret.json`（不提交到Git）：

```json
{
  "exchange": {
    "name": "binance",
    "key": "your_api_key_here",
    "secret": "your_api_secret_here"
  }
}
```

#### 方法2: 环境变量

```bash
# 设置环境变量
export BINANCE_API_KEY="your_key"
export BINANCE_API_SECRET="your_secret"
```

配置文件引用：
```json
{
  "exchange": {
    "key": "${BINANCE_API_KEY}",
    "secret": "${BINANCE_API_SECRET}"
  }
}
```

### 文件权限设置

```bash
# 敏感配置文件
chmod 600 exampleconfig_secret.json
chmod 600 config-private.json

# 日志目录
chmod 700 user_data/logs/

# 数据目录
chmod 700 user_data/data/
```

### API 权限矩阵

| 权限 | 状态 | 说明 |
|------|------|------|
| 读取账户信息 | ✅ 允许 | 必需 |
| 交易下单 | ✅ 允许 | 必需 |
| 读取订单历史 | ✅ 允许 | 必需 |
| 提现 | ❌ **禁止** | 安全风险 |
| 子账户管理 | ❌ 禁止 | 不需要 |
| API 设置修改 | ❌ 禁止 | 安全风险 |

### IP 白名单

建议在交易所设置中添加：
- VPS/服务器 IP
- 备用服务器 IP
- 不添加本地开发 IP

## 6.5 资本管理建议

### 风险管理原则

| 原则 | 建议值 | 说明 |
|------|--------|------|
| 单笔风险 | 1-2% | 账户总额的最大风险 |
| 总敞口 | < 30% | 同时持仓总值 |
| 最大开仓 | 6-12 | 并发交易数 |
| 最小资金 | $1000 | 推荐起始资金 |

### 仓位大小计算

```
单笔仓位 = 账户总额 × 风险百分比 / 止损百分比

示例:
账户: $10,000
风险: 2%
止损: 10%
单笔仓位 = $10,000 × 0.02 / 0.10 = $2,000
```

### 杠杆使用建议

| 经验级别 | 推荐杠杆 | 说明 |
|----------|----------|------|
| 新手 | 1x | 无杠杆 |
| 中级 | 2x | 温和杠杆 |
| 高级 | 3x | 标准杠杆 |
| 专业 | 3-5x | 高风险 |

## 6.8 常见错误避免

### 配置错误

| 错误 | 影响 | 解决方案 |
|------|------|----------|
| timeframe 被覆盖 | 信号异常 | 确保最后加载的配置不覆盖 |
| exit_profit_only = true | 无法止损 | 设置为 false |
| 黑名单路径错误 | 交易风险币 | 验证文件路径 |

### 运营错误

| 错误 | 影响 | 解决方案 |
|------|------|----------|
| 跳过干运行 | 真实亏损 | 必须完成2周干运行 |
| 过高杠杆 | 爆仓风险 | 从1x开始 |
| 忽略监控 | 错过问题 | 设置告警 |

### 安全错误

| 错误 | 影响 | 解决方案 |
|------|------|----------|
| 密钥提交到Git | 资金被盗 | 使用.gitignore |
| 启用提现权限 | 安全风险 | 仅限交易权限 |
| 无IP白名单 | 被攻击风险 | 设置白名单 |
